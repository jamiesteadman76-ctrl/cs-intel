# CS Intel — Seed Script

`scripts/seed/index.ts` populates the database with realistic demo data:
10 CS2 teams, 5 tournaments, 40 matches (30 to be resolved by the RPC +
10 left upcoming), 10 auth test users, ~400 predictions, and — most
importantly — a full round of `evaluate_match_predictions` calls so the
ledger (`scoring_events`) and `users.intel_score` end up in lock-step.

## Prerequisites

```bash
npm install --save-dev tsx
```

(The seed script is TypeScript; `tsx` runs it directly without a build
step. `pnpm` / `bun` work too — change the script entry in
`package.json` accordingly.)

## Required environment variables

| Variable | Notes |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Project URL, e.g. `https://abcd.supabase.co` |
| `SUPABASE_SERVICE_ROLE_KEY` | **Service-role key, NOT anon.** Used because the RPC is GRANTed only to service_role and we create auth users via `auth.admin.createUser`. |

Put them in `.env.local` (Next.js auto-loads them when invoked via tsx
under `dotenv-cli`) or export them in the shell:

```bash
export NEXT_PUBLIC_SUPABASE_URL=https://abcd.supabase.co
export SUPABASE_SERVICE_ROLE_KEY=eyJ...
```

## Run

```bash
npm run seed
```

The script logs every step with counts. Re-run any time — it is
idempotent on teams, tournaments, matches, users, and predictions.

## What gets inserted

| Object | Count | Idempotency strategy |
|---|---|---|
| `public.teams` | 10 | upsert by `slug` |
| `public.tournaments` | 5 | upsert by `slug` |
| `public.matches` | 40 | composed-key lookup on `(team1_id, team2_id, tournament_id)`; first match found is reused |
| `auth.users` + `public.users` | 10 | `auth.admin.createUser`; on duplicate email we look up via `auth.admin.listUsers` |
| `public.predictions` | 400 | upsert on `(user_id, match_id)` |
| `public.scoring_events` | per-resolve | auto-written by the RPC |
| `public.users.intel_score` | running total | auto-computed by the RPC |

## Test user credentials

After the run you can sign in as:

```
test_user_1@example.com  …  test_user_10@example.com
password: Test1234!
```

The `handle_new_user` trigger provisions `public.users` rows with
`intel_score = 0` and `is_admin = false`. Override `TEST_USERS` in
`scripts/seed/index.ts` to use different emails/passwords.

## Customisation

- **Add teams**: append to the `TEAMS` array. UUIDs are generated by
  the database; do not pre-fill them.
- **Add tournaments**: same, append to `TOURNAMENTS`.
- **Add matches**: append to `MATCH_PLAN`. `result: null` stays
  upcoming; `'team1_win' | 'team2_win' | 'draw'` is solved by the RPC
  on the run.
- **Change prediction distribution**: edit the deterministic seed
  (`rng(42)`) inside `upsertPredictions` to bias the random picks.
- **Skip resolution**: delete or comment out the `resolveCompletedMatches`
  call in `main()` to leave all matches pending.

## Notes

- The script is safe to re-run indefinitely. Each call writes
  diagnostic logs that include counts of inserted / existing rows.
- Across re-runs, match timestamps drift slightly with `Date.now()`,
  but the composed-key lookup keeps the database row count stable.
- After running, hit `verify_leaderboard_integrity()` (or wait for the
  hourly cron) to confirm `SUM(scoring_events.delta) = users.intel_score`
  for every test user.
